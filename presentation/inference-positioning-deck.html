Product narrative

Native IDE inference, not “another chat wrapper.”

RawrXD pairs a Win32-first editor with sovereign GGUF paths, agentic tools, and integrated terminals that behave like VS Code / Cursor — including read-only agent surfaces and mirrored tool execution.

Visual tone: Stripe-like structure + Anthropic-warm accent (terracotta) on key claims — not generic “AI purple slides.”
Competitor framing

Ollama vs vLLM vs RawrXD positioning

Use this table in decks and GTM; it is orthogonal axes, not “winner/loser.”

Layer Ollama vLLM RawrXD
Primary job Run models locally; simple HTTP API Serve models at scale; PagedAttention / throughput Ship a native IDE with agent + terminal + local inference
Who wins Individual devs, quick experiments Platform teams, multi-GPU serving Teams wanting Cursor-class UX without web stack in the core loop
Composable? Yes — swap models, call API Yes — OpenAI-compatible frontends Yes — bridges to routers; GGUF + tools + Win32 UI
  • Ollama is the “get a model running” layer.
  • vLLM is the “serve tokens cheaply at scale” layer.
  • RawrXD is the “edit, run, agent, prove” layer — desktop sovereign.
